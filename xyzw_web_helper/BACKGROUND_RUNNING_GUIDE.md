# XYZW Web Helper - 完全后台运行方案

## 问题解决

**你的问题：** 前端定时任务需要浏览器一直开着，无法真正后台运行。

**解决方案：** 使用 PM2 进程管理器，让前端服务在后台持续运行。

## 快速开始（小白专用）

### 1. 一键启动

```bash
# 进入项目目录
cd /workspace/xyzw_web_helper

# 运行启动脚本
./start.sh
```

### 2. 配置定时任务

1. 浏览器打开 `http://localhost:3000`
2. 进入"批量日常任务"页面
3. 添加你的游戏 Token
4. 创建定时任务（例如：每天早上8点签到）
5. **关闭浏览器** - 任务继续运行！

### 3. 设置开机自启

```bash
# 安装 PM2（如果还没装）
npm install -g pm2

# 设置开机自启
pm2 startup
pm2 save

# 按照提示执行输出的命令
```

## 工作原理

### 当前架构
```
浏览器前端（3000端口）
    ↓
定时任务调度器（前端 JavaScript）
    ↓
WebSocket 连接 → 游戏服务器
```

### 使用 PM2 后
```
PM2 后台进程
    ↓
前端服务（持续运行）
    ↓
定时任务调度器（独立运行）
    ↓
WebSocket 连接 → 游戏服务器
```

**关键点：**
- PM2 让 Node.js 服务在后台持续运行
- 即使关闭浏览器，前端服务仍在运行
- 定时任务在前端服务中独立执行
- 开机自动启动，无需手动操作

## 定时任务配置示例

### 每天固定时间执行
```
任务名称：每日签到
运行类型：每天固定时间
运行时间：08:00
```

### Cron 表达式（更灵活）
```
任务名称：每6小时领取挂机
运行类型：Cron表达式
Cron表达式：0 0 */6 * * *
```

**Cron 表达式说明：**
```
* * * * * *
│ │ │ │ │ │
│ │ │ │ │ └─ 星期几 (0-7, 0和7都是周日)
│ │ │ │ └─── 月份 (1-12)
│ │ │ └───── 日期 (1-31)
│ │ └─────── 小时 (0-23)
│ └───────── 分钟 (0-59)
└─────────── 秒 (0-59)
```

**常用示例：**
```
0 0 8 * * *      # 每天早上8点
0 30 12 * * *    # 每天中午12:30
0 0 */6 * * *    # 每6小时
0 0 20 * * 1-5   # 周一到周五晚上8点
0 0 0 1 * *      # 每月1号凌晨
```

## 故障排查

### 服务无法启动
```bash
# 检查端口占用
lsof -i:3000

# 杀掉占用进程
kill -9 <PID>

# 重新启动
./start.sh
```

### 定时任务不执行
1. 检查任务是否启用（开关状态）
2. 检查 Cron 表达式是否正确
3. 查看日志是否有错误信息
4. 确认 Token 仍然有效

### 浏览器缓存清空后
1. 使用"导入配置"恢复任务
2. 重新添加 Token
3. 重新创建定时任务

## 安全建议

1. **Token 安全**
   - Token 存储在本地，不上传服务器
   - UI 显示时隐藏敏感信息
   - 使用 WSS 加密连接

2. **网络安全**
   - 仅连接官方游戏服务器
   - 不发送敏感数据到第三方
   - 使用 HTTPS/WSS 协议

3. **数据备份**
   - 定期导出配置
   - 备份 Token 数据
   - 保存任务配置

## 常见问题

**Q: 关闭浏览器后任务会停止吗？**
A: 不会！使用 PM2 后，服务在后台持续运行。

**Q: 重启电脑后需要手动启动吗？**
A: 不需要！设置开机自启后自动启动。

**Q: 多个账号可以同时运行吗？**
A: 可以！支持批量管理多账号，最多10个并发连接。

**Q: 定时任务精度如何？**
A: 检查间隔10秒，任务会在设定时间的10秒内执行。

**Q: 可以修改端口吗？**
A: 可以！修改 `vite.config.js` 中的端口配置。
